# 外卖订单配送系统 — 完整开发规划书

> 本规划书用于指导 AI 逐步生成可运行的完整系统代码。
> 每个「步骤」对应一次独立的 AI 对话，按顺序执行。

---

## 一、系统概述

### 1.1 业务角色

| 角色 | 说明 |
|------|------|
| 消费者 | 浏览商家、点餐下单、在线支付、实时追踪配送 |
| 商家 | 管理菜单、接单处理、查看营业数据 |
| 骑手 | 抢单/接单、导航配送、确认送达 |
| 管理员 | 用户管理、商家审核、数据统计、系统配置 |

### 1.2 核心业务流程

```
消费者下单 → 支付 → 商家接单 → 平台派单给骑手
→ 骑手取餐 → 配送 → 送达确认 → 订单完成 → 评价
```

## 三、微服务架构设计

### 3.1 服务拆分

```
takeout-system/
├── takeout-gateway          # 网关服务        :8080
├── takeout-auth             # 认证服务        :8081
├── takeout-user             # 用户服务        :8082
├── takeout-merchant         # 商家服务        :8083
├── takeout-product          # 商品/菜品服务   :8084
├── takeout-order            # 订单服务        :8085
├── takeout-pay              # 支付服务        :8086
├── takeout-delivery         # 配送服务        :8087
├── takeout-notification     # 通知服务        :8088  (短信/推送)
├── takeout-search           # 搜索服务        :8089
├── takeout-file             # 文件服务        :8090
└── takeout-common           # 公共模块        (非服务，被依赖)
```

### 3.2 服务间通信

- **同步调用**: OpenFeign（服务间 REST 调用）
- **异步通信**: RocketMQ（订单状态变更、支付结果通知、配送事件）
- **实时推送**: WebSocket（订单状态实时通知给消费者/商家）

### 3.3 网关路由规则

```
/api/auth/**      → takeout-auth
/api/user/**      → takeout-user
/api/merchant/**  → takeout-merchant
/api/product/**   → takeout-product
/api/order/**     → takeout-order
/api/pay/**       → takeout-pay
/api/delivery/**  → takeout-delivery
/api/search/**    → takeout-search
/api/file/**      → takeout-file
```

### STEP 02 — 认证服务（takeout-auth）

**目标**: 实现手机号+验证码登录，下发 JWT

**向 AI 提交的 Prompt**:
```
基于上一步创建的公共模块，现在实现认证服务 takeout-auth（端口8081）。

引用模块：takeout-common
数据库：无（调用用户服务的 Feign 接口查用户）
缓存：Redis（存储验证码和Token黑名单）

需要实现：
1. pom.xml（继承父模块，引入 Spring Web, Redis, Feign, Nacos）
2. application.yml（Nacos配置中心配置、Redis配置、服务端口）
3. bootstrap.yml（Nacos地址、命名空间配置）

4. 接口实现：
   POST /api/auth/sms/send
   - 参数：phone（手机号，正则校验）
   - 生成6位随机验证码，存入 Redis（key=sms:code:{phone}，TTL=5分钟）
   - 模拟发送（打印日志，不接真实短信，预留接口）
   - 防刷：同一手机号60秒内只能发一次（用Redis setnx实现）
   - 返回 Result<Void>

   POST /api/auth/login
   - 参数：phone, code
   - 校验验证码（从Redis取，比对，验证后删除）
   - 调用用户服务 Feign 接口查询或创建用户
   - 生成 AccessToken（2小时）和 RefreshToken（7天）
   - 存入 Redis：user:token:{userId} = refreshToken
   - 返回 Result<LoginVO>（accessToken, refreshToken, userInfo）

   POST /api/auth/logout
   - 从请求头取 Authorization 解析 userId
   - 删除 Redis 中的 refreshToken
   - 返回 Result<Void>

   POST /api/auth/token/refresh
   - 参数：refreshToken
   - 校验 refreshToken 有效性（解析JWT + Redis比对）
   - 重新下发 AccessToken
   - 返回 Result<String>（新 accessToken）

5. FeignClient：UserFeignClient（调用用户服务的 getOrCreate 接口）
6. 配置类：Knife4j 接口文档配置
```

## 十、注意事项

### 给 AI 的通用附加说明（每次对话都带上）

```
通用要求：
1. Java 版本：17，使用 record 类型定义 DTO/VO
2. 所有日期使用 LocalDateTime，不用 Date
3. 金额使用 BigDecimal，不用 double
4. 接口参数用 @Valid 校验，不通过返回 Result.fail(PARAM_ERROR)
5. Service 层写业务日志（log.info）
6. 不要省略任何代码，不要用"// 省略"代替
7. 代码中的中文注释保留（帮助理解业务逻辑）
8. 每个 Controller 方法加 @Operation 注解（Swagger文档）
```

### 常见问题预防

| 问题 | 解决方式 |
|------|----------|
| 循环依赖 | 通过消息队列解耦，避免服务间双向 Feign 调用 |
| 分布式事务 | 订单+库存扣减用消息最终一致（非 XA），接受短暂不一致 |
| 接口幂等 | 支付接口用 paymentNo 唯一键防重，下单用分布式锁 |
| 缓存击穿 | 热点商家数据用 Redis 分布式锁 + 缓存空值 |
| 大并发下单 | 库存预扣在 Redis，异步同步到 MySQL |
